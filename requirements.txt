# Retail Lakehouse Pipeline

A portfolio-grade **data engineering project** that ingests raw retail files, validates them, transforms them into analytics-ready tables, and exposes KPIs through a dashboard.

## What this proves
- Python ETL design
- SQL modeling with DuckDB
- Data quality checks
- Incremental pipeline thinking
- Analytics engineering mindset
- Clean project structure recruiters can understand quickly

## Stack
- Python 3.11+
- DuckDB
- Pandas
- Streamlit
- Pytest

## Architecture
raw CSV -> ingestion -> validation -> staging tables -> mart tables -> dashboard

## Main features
- Loads orders, customers, and products from raw CSV files
- Runs validation checks for nulls, duplicates, and bad prices
- Builds warehouse-style models:
  - `stg_orders`
  - `stg_customers`
  - `stg_products`
  - `fct_sales`
  - `dim_customers`
  - `dim_products`
  - `mart_daily_revenue`
- Produces KPI metrics and trend charts in Streamlit
- Stores everything in a local DuckDB database for easy demoing
